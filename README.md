# Система безпечного розподілу секретів за схемою Шаміра (ShamirSSS)

Веб-додаток, що реалізує криптографічний протокол розділення секрету Шаміра $(k, n)$ із додатковим симетричним шифруванням даних та повним аудитом дій користувачів. Проєкт побудований на базі веб-фреймворку Django та СУБД PostgreSQL.

## ⚙️ Архітектура та технологічний стек

Проєкт спроектований за принципом розділення відповідальності (Layered Architecture), де бізнес-логіка та криптографічні обчислення винесені в окремий сервісний шар (`services.py` та `crypto/`), а представлення (Views) відповідають лише за обробку HTTP-запитів.

* **Мова програмування:** Python 3.14
* **Веб-фреймворк:** Django 6.0+ (з використанням Custom User Model)
* **Менеджер пакетів:** UV (сучасна швидка альтернатива pip)
* **База даних:** PostgreSQL
* **Криптографія:** Пакет `cryptography` (алгоритм Fernet/AES-128-CBC)
* **Фронтенд:** HTML5, Bootstrap 5 (Адаптивний інтерфейс)

## 🔒 Криптографічний базис

Безпека збереження секретів у системі базується на двох рівнях захисту:

1. **Схема розділення секрету Шаміра (Shamir's Secret Sharing):**
   Оригінальний секретний текст інтерпретується як масив байтів. Для кожного байту будується випадковий поліном ступеня $k-1$, де вільним членом є сам байт секрету. Користувачам видаються $n$ точок (часток) цього полінома. Для відновлення секрету необхідно зібрати щонайменше $k$ часток, використовуючи **інтерполяційний багаточлен Лагранжа** над скінченним полем (Поле Галуа $GF(256)$). Маючи навіть $k-1$ часток, зловмисник не отримає жодної інформації про секрет.
   
2. **Симетричне шифрування Fernet (AES-128-CBC + HMAC-SHA256):**
   Для запобігання компрометації метаданих або локального витоку часток з бази даних, згенеровані координати $Y$ додатково шифруються на рівні сервера за допомогою унікального ключа `ENCRYPTION_KEY`, який зберігається в ізольованому середовищі у файлі `.env`.

## 🗄️ Модель даних (База даних)

В системі реалізовано 4 пов'язані сутності (моделі):

* **CustomUser (Таблиця користувачів):** Розширена стандартна модель Django з використанням Email замість Username (за потребою) та зв'язком з іншими сутностями.
* **Secret (Метадані секрету):** Зберігає назву секрету, поріг $k$, загальну кількість часток $n$, власника та дату створення. *Сам секрет у цій таблиці не зберігається.*
* **Share (Частки):** Містить координату $X$, зашифровану в Fernet координату $Y$, а також зовнішні ключі на секрет та користувача-учасника.
* **AuditLog (Журнал безпеки):** Автоматично фіксує всі критичні події: реєстрацію, вхід, вихід із системи, створення секрету та спроби його відновлення (із прив'язкою до IP, таймстампу та користувача).
